【MVP 版本】UI 需求说明书（仅最小可行）
一、设计目标

支持用户完成一次完整 SCL-90 测评

序列号激活 → 答题 → 查看结果

UI 以清晰、稳定、低风险为优先

二、页面清单（MVP 仅 7 个页面）

启动页

首页 / 序列号输入页

测评说明页

测评答题页

提交确认页

测评结果页

测评记录页（限时保存）

三、页面 UI 需求
1. 启动页

居中显示产品名称

副标题：心理健康自评工具

2 秒后自动进入首页

2. 首页 / 序列号输入页

页面目标：输入序列号并进入测评

UI 元素：

产品名称

简短说明（90 题 / 约 10–15 分钟）

序列号输入框（单行，支持粘贴）

主按钮：【开始测评】

错误提示文本区域（默认隐藏）

3. 测评说明页

页面目标：告知规则并降低中途退出

UI 元素：

标题：测评说明

文本说明：

本量表共 90 题

请根据最近一周真实情况作答

免责声明文本（非诊断）

主按钮：【开始答题】

4. 测评答题页（核心）

页面结构：

顶部：

进度显示（如 12 / 90）

中部：

题目文本（大字号，居中或左对齐）

底部：

5 个选项按钮：

没有 / 很轻 / 中等 / 偏重 / 严重

交互要求：

单选后自动进入下一题

最后一题进入提交确认页

5. 提交确认页

页面目标：防止误提交

UI 元素：

提示文本：是否确认提交本次测评？

主按钮：【确认提交】

次按钮：【返回修改】

6. 测评结果页

页面目标：展示整体结果，不制造焦虑

UI 元素：

标题：测评结果

总体状态提示（文字）

总均分数值

9 个因子列表（名称 + 分数）

提示文本：结果仅供参考

次按钮：【查看历史记录】

7. 测评记录页（限时保存）

页面目标：查看一定时间内的历史测评结果

UI 元素：

标题：测评记录

列表项包含：

测评日期

总均分

总体状态标签

点击列表项进入对应测评结果页（只读）

规则提示：

页面底部提示：记录仅保存 X 天

四、通用 UI 规范（MVP）

主色：蓝色或蓝绿色

背景：白色 / 浅灰

字体层级清晰，题目字号明显大于正文

不使用图表、动画（MVP 阶段）

五、MVP UI 边界说明

不做完整账号体系（仅序列号或设备级保存）

历史记录仅保存固定时间（如 7 / 30 天，可配置）

不做跨设备同步

不做因子详情页

不做数据导出

【本说明书仅用于 MVP UI 设计与前端实现】
