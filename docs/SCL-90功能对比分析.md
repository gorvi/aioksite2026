# SCL-90 功能对比分析

## 专业报告 vs 当前实现

### ✅ 已实现的功能

1. **9个维度的平均分计算** ✅
   - 躯体化、强迫状态、人际关系敏感、抑郁、焦虑、敌对、恐怖、偏执、精神病性
   - 计算方法：维度总分 ÷ 维度题目数

2. **总均分计算** ✅
   - 计算方法：90题总分 ÷ 90

3. **简单的状态判断** ✅
   - < 2.0：心理状态整体稳定
   - 2.0-2.9：存在一定心理压力
   - ≥ 3.0：心理困扰较明显

### ❌ 缺失的功能

#### 1. **原始分（总分）**
- **专业报告**：显示 90 题的总分（如：351 分）
- **当前实现**：只计算总均分，不显示原始总分
- **实现难度**：简单
- **建议**：在结果中增加 `raw_total_score` 字段

#### 2. **阳性/阴性项目数**
- **专业报告**：
  - 阳性项目数：得分 ≥ 2 的题目数量
  - 阴性项目数：得分 < 2 的题目数量
- **当前实现**：未计算
- **实现难度**：简单
- **建议**：增加 `positive_items_count` 和 `negative_items_count` 字段

#### 3. **阳性项目平均分**
- **专业报告**：只计算阳性项目的平均分
- **当前实现**：未计算
- **实现难度**：简单
- **建议**：增加 `positive_items_average` 字段

#### 4. **参考等级（轻/中/重/极重）**
- **专业报告**：基于常模数据，给出每个维度的参考等级
- **当前实现**：只有简单的稳定/压力/明显三个等级
- **实现难度**：中等（需要常模数据）
- **建议**：
  - 需要 SCL-90 中国常模数据（均分±标准差）
  - 根据得分与常模的对比，判断等级
  - 等级划分：< 1.5 轻，1.5-2.5 中，2.5-3.5 重，≥ 3.5 极重

#### 5. **均分±标准差（常模参考）**
- **专业报告**：显示每个维度的常模参考值（如：1.37±0.48）
- **当前实现**：无
- **实现难度**：中等（需要常模数据）
- **建议**：需要获取 SCL-90 中国常模数据

#### 6. **折线图可视化**
- **专业报告**：用折线图展示各维度得分趋势
- **当前实现**：只有卡片展示
- **实现难度**：中等
- **建议**：使用图表库（如 Chart.js、Recharts）绘制折线图

#### 7. **详细测评结论**
- **专业报告**：基于具体得分给出诊断性描述（如："重度的躯体化症状"）
- **当前实现**：只有简单的状态描述
- **实现难度**：中等
- **建议**：
  - 根据各维度得分和参考等级，生成详细结论
  - 可以基于规则引擎生成描述性文本

## 实现建议

### 优先级 1：基础统计（容易实现）

1. **添加原始分（总分）**
   ```typescript
   raw_total_score: number; // 90题总分
   ```

2. **添加阳性/阴性项目统计**
   ```typescript
   positive_items_count: number; // 得分 ≥ 2 的题目数
   negative_items_count: number; // 得分 < 2 的题目数
   positive_items_average: number; // 阳性项目平均分
   ```

### 优先级 2：参考等级（需要常模数据）

3. **添加参考等级判断**
   - 需要 SCL-90 中国常模数据
   - 根据得分与常模对比，判断等级（轻/中/重/极重）

4. **显示常模参考值**
   - 显示每个维度的均分±标准差

### 优先级 3：可视化（可选）

5. **添加折线图**
   - 使用图表库绘制各维度得分趋势

6. **生成详细结论**
   - 基于得分和等级，生成描述性结论

## 结论

**当前实现可以：**
- ✅ 计算基本的得分和平均分
- ✅ 给出简单的状态判断
- ✅ 展示各维度得分情况

**当前实现不能：**
- ❌ 提供专业的参考等级判断（需要常模数据）
- ❌ 显示详细的统计信息（阳性项目数等）
- ❌ 生成诊断性的详细结论
- ❌ 提供可视化图表

**如果要达到专业报告的效果，需要：**
1. 获取 SCL-90 中国常模数据（均分±标准差）
2. 实现参考等级判断逻辑
3. 添加更多统计指标
4. 实现可视化图表
5. 开发结论生成引擎

**注意：** 根据合规要求，我们只能提供"自我了解"工具，不能提供医学诊断。因此，即使实现了上述功能，结论描述也需要符合合规要求。
